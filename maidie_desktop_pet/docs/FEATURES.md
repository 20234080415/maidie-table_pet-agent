# 功能说明

## 按需千问视觉

- 明确的看屏幕、看窗口、看报错或看题目请求会进入 `qwen3-vl-flash` 视觉流程。
- 模糊指代会先确认，不会直接截图；普通聊天不触发截图。
- 视觉结果是结构化数据，最终回复由 DeepSeek 和 Synthesizer 生成。
- 截图在内存中压缩，短缓存只保存结构化结果。
- 本地定时 OCR 与按需千问视觉是两条独立链路。

配置、测试和当前限制见[千问视觉与屏幕理解](VISION.md)。

## 桌面窗口

- 无边框、透明背景、始终置顶的工具窗口。
- 默认尺寸可配置，支持边缘拖拽、滚轮和右下角控制点缩放。
- 角色按原始 `192×208` 帧和高 DPI 画布重新渲染，避免累计缩放模糊。
- 气泡和输入框是独立跟随浮窗，会避开角色和屏幕边缘。
- 右下角缩放控制点只在鼠标位于角色窗口时显示。

## 动画系统

主图集采用 hatch-pet `1536×1872`、8 列 × 9 行 WebP 格式，每格 `192×208`。状态包括：

`idle`、`walk-left/right`、`run-left/right`、`thinking`、`talking`、`waiting`、`review`、`failed`、`happy`、`reacting`、`sleeping`。

动作切换包含约 160ms 交叉过渡。外部动作可在 `assets/actions/actions.json` 中独立配置帧率、持续时间、冷却、优先级和触发词，详见[动作系统](ACTIONS.md)。

## 移动与自主行为

- 独立移动控制器管理速度、加速度、目标点和屏幕边界。
- 根据速度自动切换 `idle`、`walk` 和 `run`。
- 支持自主游走、目的性停顿、屏幕边缘避让和可选光标追逐。
- 自主停顿时可能播放瞌睡动作。
- 主动行为默认关闭，并使用全局冷却避免频繁打扰。

## 鼠标与键盘互动

- 点击头顶：摸头反应，不调用 AI。
- 在头顶左右往返滑动：连续抚摸。
- 点击脸颊：戳脸反应。
- 点击身体：触发自然回复。
- 明显拖动：移动 Maidie；向右拖动超过阈值可播放晕乎动作。
- 双击 Maidie，或按 Enter / 空格：打开聊天输入框。
- Esc、失焦、发送完成或 10 秒无输入后收起输入框。

## 围栏模式

右键 Maidie 可开启或解除围栏模式。开启后：

- 显示低透明度圆角围栏边框。
- 拖动顶部中央把手可移动围栏。
- 拖动四边或四角可改变大小。
- 围栏中央保持点击穿透，只有细边框接收编辑输入。
- 最终范围会限制在当前屏幕内，并保证至少能够容纳 Maidie。
- 如果调整后 Maidie 位于范围外，会回到最近的有效位置。
- 自动游走、拖拽和 snapback 使用同一最终围栏范围。

可通过 `fence.show_overlay` 关闭边框显示，不影响围栏移动约束。

## AI 与聊天

- 支持 DeepSeek / OpenAI Chat Completions 兼容接口。
- SSE 流式文本经过分句和节奏控制后逐段显示。
- 日常交流使用聊天路由；代码、构建、调试和技术文档问题使用技术路由。
- 时间、天气、搜索、屏幕和记忆等事实通过工具取得，不由模型猜测。
- 完整 Agent 设计见[技术架构](TECHNICAL_OVERVIEW.md)。

## 人格与记忆

- 提供温柔傲娇、元气活泼、安静治愈、优雅女仆和自定义人格。
- 当前使用 `memory/memories.db` SQLite 数据库。
- 保存最多 20 条近期聊天，并支持长期事实和偏好。
- 对话结束后可在后台提取事实和偏好；缺少可用模型 Key 时仍可保存聊天，但不执行长期记忆提取。
- 敏感信息过滤和数据库边界见[隐私与安全](PRIVACY_AND_SAFETY.md)。

## 桌面感知与主动行为

- 鼠标活动、空闲时长、前台窗口、应用类型和可选 OCR 组成桌面上下文。
- 主动行为支持久未活动、长时间编程、场景变化和频繁切换窗口等触发源。
- 主动行为默认关闭；检查间隔和全局冷却可配置，避免频繁打扰。
- `once`、`cron` 和 `condition` 任务保存在本地 `memory/scheduled_tasks.json`。
- 感知内容和默认开关见[隐私与安全](PRIVACY_AND_SAFETY.md)。

## 状态与优先级

中央状态机是唯一状态来源，状态只能通过 `PetController` 协调。

优先级从高到低为：

1. 用户点击、摸头、戳脸和拖动。
2. 光标互动。
3. AI 思考和说话。
4. 自主游走、停顿和瞌睡。
5. 普通待机。

## 常见问题

### 填了 Key 仍无法聊天

确认 Base URL 和模型名称；检查 `logs/maidie.log`；确认服务支持 `/chat/completions` 和 SSE。若设置了 `DEEPSEEK_API_KEY`，它会覆盖 JSON 中的主 Key。

### 气泡或输入框没有跟随

重启 Maidie 以加载最新代码。两者都是独立跟随浮窗，会在角色移动时重新定位。

### 动作没有播放

确认动作文件存在于 `assets/actions/`，文件名、帧数与 `actions.json` 一致，并检查动作冷却时间。

### 放大后角色变柔和

程序始终从原始帧重新渲染，不会累积低清缓存。超过素材原生分辨率后的柔化属于源图限制，需要更高分辨率素材或其他渲染方案。
