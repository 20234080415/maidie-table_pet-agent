# 隐私与安全边界

Maidie 采用“敏感能力默认关闭、按任务提供最少上下文、写操作显式确认、危险执行始终禁止”的原则。

## 数据会流向哪里

| 能力 | 可能发送的数据 | 处理方 |
|---|---|---|
| AI 聊天 | 用户问题、必要对话上下文、结构化工具结果 | 用户配置的模型服务 |
| Tavily 搜索 | 为完成请求所需的查询词 | Tavily |
| Qwen VL | 用户明确选择范围内的压缩截图和问题 | 阿里云百炼 / DashScope |
| 本地 OCR | 截图像素；识别文字可能作为必要上下文进入后续 AI 请求 | 本机 Tesseract；必要时模型服务 |

第三方服务可能按各自政策记录请求。不要在问题、截图或查询中包含不必要的密码、密钥、支付信息或私人通信。

## 屏幕理解

- Qwen VL 只在明确的屏幕、窗口、图片或区域请求后运行；模糊请求先确认。
- 截图在内存中缩放并编码为 JPEG，再发送到用户配置的阿里云百炼 Workspace。
- 原始截图不写入磁盘；短缓存保存结构化视觉结果，不保存原图。
- 不会因为当前窗口读取失败而静默扩大到全屏范围。
- Qwen VL 只提取结构化事实，最终回答由 Synthesizer 生成。

## OCR

- 本地 OCR 默认关闭，启用后由本机 Tesseract 处理。
- OCR 与按需 Qwen VL 独立，不互相作为自动降级。
- OCR 识别结果可能含屏幕文字；启用前应确认当前桌面不包含不希望处理的信息。

## 窗口、应用和剪贴板

- 窗口感知可能读取前台窗口标题、进程名、窗口类和活动类型。
- 剪贴板默认只检测“是否发生变化”，不读取正文。
- 只有用户明确要求处理刚复制的内容，或经过对应确认流程时，才应读取必要正文。
- 主动行为默认关闭；即使启用，也不应因剪贴板变化直接外发正文。

## 本地记忆

记忆保存在 `memory/memories.db` SQLite 数据库：

- 近期聊天最多保留 20 条；长期记忆分为事实和偏好。
- 密码、API Key、令牌、证件、银行卡、联系方式、地址和健康隐私等会经过规则过滤。
- 时间事件短期上下文只存在于当前 Router 会话，不写入 SQLite。
- 右键菜单可清除近期聊天、长期事实和偏好。

禁止提交：

```text
memory/*.db
memory/*.db-wal
memory/*.db-shm
memory/conversations.json
memory/scheduled_tasks.json
```

## 系统工具安全边界

允许的只读或受限操作包括文件读取、受限文件搜索和截图保存。创建文件、打开应用、打开目录、切换窗口和写入剪贴板必须经过可信确认流程；确认框默认选择“否”，超时或无法确认均按拒绝处理。

Planner 输出不等于授权。Executor 和 `SystemTool` 会重新校验操作与参数。

始终禁止：

```text
delete_file
execute_script
system_command
任意 shell 执行
```

Maidie 因此不是通用命令执行器，也不会接受 Planner 伪造的 `confirmed=true` 绕过确认。

## API Key

- 设置界面使用密码输入框，但 `config/config.json` 中仍是本地明文。
- 推荐使用 `DEEPSEEK_API_KEY`、`DASHSCOPE_API_KEY` 等环境变量。
- 不要提交 `config/config.json`、`.env`、日志、截图或含真实 Key 的示例。
- 公共设置快照只暴露是否已配置，不返回 Key 正文。

## 日志和问题报告

日志可能包含路由类型、工具状态、窗口或视觉处理元数据，但不应记录 API Key。报告安全问题前，应移除真实凭据、私人截图、数据库、用户路径和复现数据中的隐私内容。
