# 隐私与安全边界

## 千问视觉隐私边界

- 只有用户明确请求查看屏幕、窗口或图片时才调用千问视觉，不进行每秒截图或静默上传。
- 截图会在内存中缩小并编码，然后发送到用户配置的阿里云百炼业务空间。
- 原始截图不写入磁盘，也不进入永久缓存；默认 5 秒缓存只保存结构化视觉结果。
- 日志记录截图范围、尺寸、压缩大小、任务类型、置信度和缓存命中状态，不记录 API Key。
- 设置界面中的视觉 Key 使用密码输入框；公共设置快照只暴露是否已配置。但写入 `config/config.json` 时仍是本地明文，推荐使用 `DASHSCOPE_API_KEY` 环境变量。
- 千问视觉只提取结构化事实，DeepSeek 接收用户问题和这些事实以生成最终回答。

不要在包含密码、密钥、支付信息或私人通信的窗口上触发屏幕理解。完整说明见[千问视觉与屏幕理解](VISION.md)。

Maidie 包含屏幕理解、联网、记忆和受控系统工具。项目采用“默认关闭高敏感感知、写操作显式确认、危险执行始终禁止”的原则。

## 屏幕理解与 OCR

- OCR 默认关闭，只有用户启用后才会通过本机 Tesseract 识别屏幕。
- 截图在内存中交给 OCR 处理；识别文字可能随当前 Agent 任务发送到用户配置的 AI 服务。
- OCR 质量依赖语言包、屏幕缩放和画面清晰度。
- WindowTracker 会忽略 Maidie 自身窗口；围栏 overlay 不接受焦点，不应成为用户当前窗口。

## 窗口、应用和剪贴板感知

- 窗口和应用跟踪读取前台窗口标题、进程名、窗口类和活动类型。
- 剪贴板追踪只检测内容是否变化，不读取剪贴板正文。
- 主动行为默认关闭；启用后可根据空闲、窗口切换、场景变化或剪贴板变化发出提示。

## 系统工具权限

只读操作：

- `read_file`：读取文件，单文件限制 2 MB。
- `search_files`：按限制数量搜索文件。
- `screenshot`：保存截图；默认拒绝覆盖已有文件。

需要 PyQt 确认框的操作：

- `create_file`
- `open_app`
- `open_folder`
- `switch_window`
- `copy_clipboard`

确认框默认选择“否”；超时或无法确认时按拒绝处理。Planner 提供的 `requires_confirmation` 不可信，执行层会重新执行安全判断。

始终禁止：

- `delete_file`
- `execute_script`
- `system_command`

这些限制意味着 Maidie 不是任意命令执行器或通用桌面自动化框架。

## 联网查询

联网功能默认关闭。启用后，当前问题和必要查询词会发送给所选第三方搜索服务；服务商可能按其隐私政策记录请求。Maidie 不会为了搜索自动上传本地近期聊天记录。

搜索 API Key 以明文保存在本地 `config/config.json`。如果不能接受第三方处理查询，请保持联网开关关闭。

## AI 服务与 API Key

- AI 请求会发送用户问题、必要上下文和工具结果到配置的模型服务。
- 设置界面的 Key 输入框采用密码显示，公共设置快照只暴露“是否已配置”。
- JSON 配置中的 Key 仍是本地明文；推荐通过 `DEEPSEEK_API_KEY` 环境变量提供。
- 不要把真实 Key、令牌、密码或本地配置提交到公开仓库。

## 记忆系统

当前记忆使用本地 SQLite：`memory/memories.db`。

- 最多保存 20 条近期 `chat` 对话。
- 长期记忆分为 `fact` 和 `preference`。
- 每轮对话结束后，可在后台使用当前聊天模型提取事实和偏好。
- API Key、密码、令牌、证件、银行卡、联系方式、地址和健康隐私等内容会被规则过滤，不写入长期记忆。
- 提取失败或数据库暂不可用时自动降级，不影响聊天继续运行。
- 右键菜单中的“清除记忆”会清除近期聊天和长期事实、偏好。

`memories.db` 及其 WAL/SHM 文件被 `.gitignore` 排除，只保存在本机。不要手动将这些数据库文件加入提交。

## 安全报告建议

发现可能泄露凭据、绕过确认或执行任意命令的问题时，不要在公开 issue 中附带真实 Key、私人截图、数据库或复现数据；先移除敏感信息再报告。
